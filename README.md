# Chess Game Collection / 棋类游戏合集

一个基于 **C++17、Visual Studio 2022、EasyX、JavaScript 与 Python** 构建的本地棋类游戏平台。项目同时提供：

- **本地 C++ 版 v1.3.4**：使用 C++17、EasyX 和原生外部棋力引擎；
- **本地网站版 v1.4**：使用浏览器前端、Python 本地服务、账号系统、逐局存档和历史记录。

平台整合了 **中国象棋、国际象棋、围棋、五子棋、飞行棋和跳棋** 六种游戏。桌面版通过统一大厅进入不同棋类；网站版通过浏览器登录后进入六棋大厅，并为每个账号独立保存对局记录和棋局状态。

> **特别说明：国际象棋、围棋和五子棋的桌面版前置基础代码已经统一。**  
> 三类棋共同复用 `daima/qilei_common.h/.cpp` 中的设置界面、步时与总时、日志、机器人模式、难度换算、项目路径定位和外部引擎进程通信。各棋类仅保留自己的棋盘规则、协议转换、局势分析、后备机器人和专项界面。

## 2. 双版本说明

### 2.1 本地 C++ 版 v1.3.4

桌面版以 EasyX 图形窗口运行，适合课程设计、C++ 图形编程和本地桌面体验。

主要特点：

- 六种棋类共用一个解决方案和一个程序入口；
- 统一欢迎大厅；
- 鼠标和键盘双操作；
- 每步时长与单方总时长；
- 真人对战、单方机器人和双方机器人；
- 外部棋力引擎与内置后备算法；
- 局势分析；
- 悔棋、重开、投降或返回大厅；
- 飞行棋、跳棋支持多人完整排名；
- 游戏日志统一保存到根目录 `result/`。

### 2.2 本地网站版 v1.4

网站版不是把 EasyX 可执行文件嵌入网页，而是使用 JavaScript 重新实现六种棋的浏览器界面和主要规则。

主要特点：

- 双击本地启动器后在浏览器运行；
- 默认仅监听 `127.0.0.1`；
- 纯数字账号注册和登录；
- 用户独立存档；
- 历史记录；
- 个人改密；
- 管理员用户管理；
- 浏览器 Canvas 棋盘；
- 自动保存和读取上次存档；
- 四类棋通过 Python 后端调用根目录外部引擎；
- 引擎不可用时回退到浏览器内置机器人；
- 飞行棋、跳棋使用浏览器内置多人机器人。

## 3. 特别说明：三类棋共用统一前置代码

国际象棋、围棋和五子棋的桌面版本共同依赖：

```text
daima/qilei_common.h
daima/qilei_common.cpp
```

### 3.1 统一设置

三类棋共同使用：

```cpp
struct qilei_game_setting
{
    int qilei_step_seconds;
    int qilei_total_seconds;
    int qilei_robot_mode;
    int qilei_robot_level;
};
```

统一设置内容包括：

| 设置项 | 可选值 |
| --- | --- |
| 每步时长 | 30 秒、60 秒、90 秒 |
| 单方总时长 | 5 分钟、10 分钟、15 分钟、20 分钟 |
| 对战模式 | 双方玩家、先手机器人、后手机器人、双方机器人 |
| 机器人难度 | 入门、进阶、大师 |

### 3.2 统一计时

三类棋共同使用 `qilei_clock`：

- 分别保存双方的本步剩余时间；
- 分别保存双方总剩余时间；
- 只扣除当前行棋方时间；
- 成功走棋后切换行动方；
- 步时或总时耗尽时返回超时方；
- 引擎启动准备完成后才正式开始计时；
- 支持格式化为 `mm:ss`。

### 3.3 统一日志

三类棋共同使用 `qilei_result_logger`：

- 自动定位项目根目录；
- 在根目录 `result/` 中创建日志；
- 文件名包含棋类和时间；
- 使用 UTF-8 with BOM；
- 记录设置、走棋、机器人、分析、悔棋、投降、超时和结果。

### 3.4 统一引擎进程

三类棋共同使用 `qilei_engine_process`：

- 创建 Windows 匿名管道；
- 使用 `CreateProcessW()` 启动隐藏窗口子进程；
- 写入协议命令；
- 读取标准输出；
- 等待协议标记；
- 处理超时；
- 退出时释放进程、线程和管道句柄。

| 棋类 | 引擎 | 协议 |
| --- | --- | --- |
| 国际象棋 | Stockfish 18 | UCI |
| 围棋 | KataGo 1.16.5 | GTP |
| 五子棋 | Rapfi 250615 | Piskvork / Gomocup |

### 3.5 统一不等于规则相同

三类棋仍然分别实现：

- 棋盘数据结构；
- 合法走法；
- 胜负和和棋规则；
- 协议坐标转换；
- 局势分析；
- 棋盘绘制；
- 后备机器人。

项目结构可概括为：

```text
统一设置、计时、日志、引擎进程
                  +
国际象棋 / 围棋 / 五子棋各自规则
```

## 4. 六种棋类

### 4.1 中国象棋

- 标准 10 × 9 棋盘；
- 七类棋子完整基础规则；
- 九宫、塞象眼、蹩马腿、炮架、过河和将帅照面；
- 过滤送将走法；
- 将军、吃帅和无合法走法判定；
- Pikafish + NNUE 机器人；
- 机器人走法本地二次校验；
- 计时、局势分析、悔棋和日志。

### 4.2 国际象棋

- 标准 8 × 8 棋盘；
- 王、后、车、象、马、兵；
- 王车易位；
- 吃过路兵；
- 小兵升变；
- 将军、将死、逼和和五十步规则；
- Stockfish 18；
- 外部引擎失败时使用本地 alpha-beta 后备机器人；
- 提示、悔棋、投降和局势分析。

### 4.3 围棋

- 标准 19 路棋盘；
- 连通块和气；
- 提子；
- 禁止自杀；
- 简单劫；
- 停一手；
- 双方连续停一手结束；
- 中国数子法；
- 白方 7.5 目贴目；
- KataGo；
- 外部引擎失败时使用本地提子启发式机器人。

### 4.4 五子棋

- 标准 15 × 15 棋盘；
- 无禁手自由规则；
- 横、纵、两条斜线五连；
- Rapfi；
- 外部引擎失败时使用连型评分后备机器人；
- 活三、冲四等局势分析；
- 二次落点确认。

### 4.5 飞行棋

- 2–6 名玩家；
- 每人 2–6 个棋子；
- 0 至全部玩家机器人；
- 60 格正六边形公共跑道；
- 基地、起点、终点列和停车区；
- 抽签、掷 6 起飞和额外回合；
- 同色飞跃、黑色陷阱和对方起点暂停；
- 每步/单方计时；
- 超时淘汰；
- 完整排名；
- 快照式撤销。

### 4.6 跳棋

- 标准 121 孔星形棋盘；
- 2–6 名玩家；
- 每人 3、6 或 10 个棋子；
- 相邻一步移动；
- BFS 连续跳跃；
- 机器人推进、撤离和反循环策略；
- 自己最终区只进不出；
- 其他玩家最终区撤离规则；
- 最终区域保护；
- 超时和违规淘汰；
- 完整排名。

## 8. 桌面版运行方法

1. 安装 Visual Studio 2022；
2. 安装“使用 C++ 的桌面开发”；
3. 安装 EasyX；
4. 打开：

```text
chessgame.sln
```

5. 选择：

```text
Debug | x64
```

或：

```text
Release | x64
```

6. 确认根目录 `engines/` 完整；
7. 点击“本地 Windows 调试器”。

工程使用：

```text
C++17
Unicode
/utf-8
```

平台工具集为：

```text
v145
```

本地没有该工具集时，可在项目属性中切换到已安装的 MSVC 工具集。

## 9. 网站版运行方法

推荐双击：

```text
website/exe/chessgame_web.exe
```

默认地址：

```text
http://127.0.0.1:8765
```

端口占用时，程序会向后寻找可用端口。

也可以运行：

```powershell
python website/daima/backend/qilei_server.py
```

不自动打开浏览器：

```powershell
python website/daima/backend/qilei_server.py --no-browser
```

为了使用四类棋外部引擎，应保持以下相对结构：

```text
项目根目录/
├── engines/
└── website/
```

只移动 `website` 时，网站主体仍可运行，但外部引擎将无法找到，相关棋类会使用浏览器后备机器人。

## 13. 已知限制

### 13.1 网站定位

网站默认是本机服务，不是公有云系统：

- 默认无 HTTPS；
- 会话保存在服务器进程内存；
- 重启服务后需要重新登录；
- 默认管理员凭据必须首次使用后修改；
- 不建议直接绑定公网地址。

### 13.2 外部引擎环境

不同引擎可能依赖：

- Windows x64；
- AVX2、SSE4.1 或 POPCNT；
- VC++ 运行库；
- 模型和配置文件。

不兼容时，国际象棋、围棋和五子棋可回退；中国象棋桌面机器人需要 Pikafish 正常运行。

### 13.3 规则范围

- 中国象棋未覆盖完整竞赛和棋规则；
- 国际象棋可继续补充三次重复和不足子力；
- 围棋当前为简单劫；
- 五子棋当前为无禁手规则；
- 飞行棋击退规则仍可扩展；
- 跳棋最终区保护属于项目自定义规则。
